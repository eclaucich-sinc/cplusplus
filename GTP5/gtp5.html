Ejercicio 1

NoteConsigna

Modificar el ejercicio 1 de la guía anterior (figuras geométricas) agregando una función label() genérica que imprima el nombre dinámico del objeto utilizando la herramienta typeid de RTTI.

En C++, el polimorfismo permite tratar clases derivadas a través de una referencia o puntero a su clase base. La herramienta typeid de la librería <typeinfo> permite consultar el tipo de dato subyacente exacto en tiempo de ejecución (Run-Time Type Identification).

#include <typeinfo>

void label(geoshape_t &g) {
    cout << "Nombre del objeto (RTTI): " << typeid(g).name() << endl;
}

Al invocar esta función sobre objetos derivados instanciados, como square_t o circle_t, el operador evalúa el tipo dinámico e imprime el nombre resultante de la decoración de variables (name mangling) generada por el compilador (por ejemplo, el compilador g++ antepone al nombre original de la clase la cantidad de caracteres que este posee).

cout << "--- Cuadrado (L=2) ---" << endl;
square_t sq(-1.0, -1.0, 1.0, 1.0);
label(sq);

cout << "--- Circulo (R=2) ---" << endl;
circle_t circ(0.0, 0.0, 2.0);
label(circ);

cout << "--- Elipse (rx=2, ry=1) ---" << endl;
ellipse_t el(0.0, 0.0, 2.0, 1.0);
label(el);

Ejercicio 2

NoteConsigna

Implementar la función void diffsym(const vector<set<int>> &vecset, set<int> &output) que calcule la diferencia simétrica generalizada de un arreglo de \(N\) conjuntos empleando la estructura de datos asociativa std::map<int, int>.

La diferencia simétrica generalizada consta en obtener únicamente aquellos elementos que están presentes en exactamente un solo conjunto a lo largo de toda la colección. El uso de std::map simplifica la resolución de este problema algorítmico, ya que permite iterar a través de todos los conjuntos e ir contando las ocurrencias independientes de cada valor en un solo ciclo acumulativo:

void diffsym(const vector<set<int>>& vecset, set<int>& output) {
    map<int, int> count;
    
    // Contar apariciones globales
    for (const auto& s : vecset) {
        for (int elem : s) {
            count[elem]++;
        }
    }
    
    // Filtrar diferencia simétrica (frecuencia estricta igual a 1)
    output.clear();
    for (const auto& pair : count) {
        if (pair.second == 1) {
            output.insert(pair.first);
        }
    }
}

El resultado de las pruebas para tres conjuntos solapados (A={1,2,3,4}, B={3,4,5,6}, C={1,6,7}) demuestra su efectividad:

Ejercicio 3

NoteConsigna

Reescribir el ejemplo de los vectores (vector_t) de la GTP4 utilizando templates, de forma tal que se puedan instanciar vectores dinámicos (fullvec_t y sparse_t) de cualquier tipo primitivo (incluyendo clases que tengan sobrecargado su operador + y < como el objeto std::string).

El sistema polimórfico de clases abstractas desarrollado en prácticas previas se reescribió en este ejercicio sustituyendo el tipo numérico rígido y estricto (double) por un parámetro de plantilla paramétrica typename T:

template <typename T>
class vector_t {
    public:
        virtual int size() const = 0;
        virtual T operator[](int i) const = 0;
};

Las funciones auxiliares globales que evalúan la jerarquía también fueron transformadas a funciones template (sum, non_null, max, min y reduce), las cuales logran deducir el tipo subyacente y devuelven este mismo tipo paramétrico original. Para suplir la carencia algorítmica del número nulo absoluto 0.0 en operaciones aritméticas, las nuevas funciones asumen instanciar dinámicamente un constructor vacío T() como valor neutro de inicio (ej: 0 para enteros y "" para strings).

template <typename T>
T sum(const vector_t<T> &v) {
    if (v.size() == 0) return T();
    T total = v[0];
    for (int i = 1; i < v.size(); ++i) {
        total = total + v[i];
    }
    return total;
}

A través de esta migración estructural, el mismo código fuente de clases abstractas y métodos algorítmicos se recicla completamente, pudiendo utilizarse tanto para operaciones analíticas vectoriales estándar como para realizar la concatenación continua de palabras dentro de un vector de strings:

// Prueba con double
fullvec_t<double> fv_d({1.0, 0.0, -3.0, 4.0, 0.0});
cout << "--- Full Vector <double> ---" << endl;
cout << "Non nulls: " << non_null(fv_d) << " | Sum: " << sum(fv_d) << " | Max: " << max(fv_d) << " | Min: " << min(fv_d) << endl;

// Prueba con string (operator+ concatena, > y < son alfabéticos)
fullvec_t<string> fv_s({"Hola", "", "Mundo", "!", ""});
cout << "\n--- Full Vector <string> ---" << endl;
cout << "Non nulls: " << non_null(fv_s) << " | Sum: " << sum(fv_s) << " | Max: " << max(fv_s) << " | Min: " << min(fv_s) << endl;

// Prueba Sparse con int
sparse_t<int> spv_i(10, {2, 5, 8}, {100, -50, 25});
cout << "\n--- Sparse Vector <int> ---" << endl;
cout << "Non nulls: " << non_null(spv_i) << " | Sum: " << sum(spv_i) << " | Max: " << max(spv_i) << " | Min: " << min(spv_i) << endl;

// Prueba Reduce con string
sum_t<string> string_sum_op;
cout << "\n--- Prueba Reduce (Full Vector <string>) ---" << endl;
cout << "Reduce Sum: " << reduce(fv_s, string_sum_op) << endl;

Ejercicio 4

NoteConsigna

A partir del vector genérico de punteros vector<player_t*> players de la guía anterior (simulador Tic-Tac-Toe), iterar a todos los agentes creados al azar y filtrar con exclusividad aquellos que correspondan a la clase derivada dumb_t. Hacerlo utilizando dos variantes del RTTI: dynamic_cast con punteros y dynamic_cast con referencias (involucrando el uso de excepciones).

El operador dynamic_cast evalúa en tiempo de ejecución si la conversión (downcasting) entre un objeto polimórfico padre e hijo en un árbol jerárquico es válida o no.

En la primera implementación iterativa por medio de punteros, una falla en el proceso de casting da como resultado la asignación inofensiva de una región nula de memoria (nullptr), por lo que el uso de un condicional lógico simple resuelve el problema fácilmente:

cout << "--- Filtrando dumb_t usando dynamic_cast<> con PUNTEROS ---" << endl;
for(player_t* p : tournament) {
    dumb_t* dumb_ptr = dynamic_cast<dumb_t*>(p);
    if (dumb_ptr != nullptr) {
        cout << "Encontrado: " << dumb_ptr->label() << endl;
    }
}

Por otro lado, cuando se obtiene una referencia nula, al tratar de desreferenciarla, el lenguaje se encarga de abortar y alertar sobre la operación errónea lanzando una excepción std::bad_cast estandarizada. Para tener en cuenta estos casos, se implementa un bloque de excepción try-catch.

cout << "\n--- Filtrando dumb_t usando dynamic_cast<> con REFERENCIAS y EXCEPCIONES ---" << endl;
for(player_t* p : tournament) {
    try {
        dumb_t& dumb_ref = dynamic_cast<dumb_t&>(*p);
        cout << "Encontrado: " << dumb_ref.label() << endl;
    } catch (const bad_cast& e) {
        // Ignorado silenciosamente: el objeto no recae en la clase dumb_t
    }
}

Probando un ejemplo con varios jugadores de distintos tipos:

srand(time(NULL));
vector<player_t*> tournament;

// Generamos 8 jugadores de tipos aleatorios
for(int i = 1; i <= 8; i++) {
    int r = rand() % 5;
    string id = to_string(i);
    if (r == 0) tournament.push_back(new dumb_t(id));
    else if (r == 1) tournament.push_back(new random_t(id));
    else if (r == 2) tournament.push_back(new agressive_t(id));
    else if (r == 3) tournament.push_back(new minimax_t(id));
    else tournament.push_back(new rule_based_t(id));
}

// Imprimimos la lista completa solo para contexto
cout << "=== Lista completa de jugadores generados ===" << endl;
for(player_t* p : tournament) {
    cout << p->label() << endl;
}
cout << "=============================================\n" << endl;

// 1er Método: Usando dynamic_cast<> con punteros
cout << "--- Filtrando dumb_t usando dynamic_cast<> con PUNTEROS ---" << endl;
for(player_t* p : tournament) {
    dumb_t* dumb_ptr = dynamic_cast<dumb_t*>(p);
    if (dumb_ptr != nullptr) {
        cout << "Encontrado: " << dumb_ptr->label() << endl;
    }
}

cout << "\n--- Filtrando dumb_t usando dynamic_cast<> con REFERENCIAS y EXCEPCIONES ---" << endl;
for(player_t* p : tournament) {
    try {
        // Se desreferencia p para obtener una referencia
        dumb_t& dumb_ref = dynamic_cast<dumb_t&>(*p);
        cout << "Encontrado: " << dumb_ref.label() << endl;
    } catch (const bad_cast& e) {
        // El objeto subyacente no es un dumb_t, la excepción es capturada silenciosamente.
    }
}

// Limpieza de memoria
for(player_t* p : tournament) {
    delete p;
}
