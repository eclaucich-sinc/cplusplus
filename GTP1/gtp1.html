Ejercicio 1: Recursión

NoteConsigna

Utilizar una variable global NCALLS para contar el número de llamadas a la función intpow realizada en el GTP0.

Cada vez que se llama a intpow, se incrementa NCALLS y se imprime su valor. Luego, se ejecuta el programa para calcular 1^100, 1^1000 y 1^10000 y se observa el número de llamadas realizadas en cada caso.

La cantidad de operaciones esperadas en esta implementación recursiva de potencia es del orden de O(log p), donde p es el exponente. Esto se debe a que en cada llamada recursiva, el exponente se reduce aproximadamente a la mitad (en el caso de exponentes pares) o a la mitad menos uno (en el caso de exponentes impares). Por lo tanto, el número de llamadas a la función crecerá logarítmicamente con respecto al valor del exponente.

Para demostrar esto, se ejecuta el cálculo de \(1^{100}\), \(1^{1000}\) y \(1^{10000}\), y se observa el número de llamadas realizadas en cada caso. Como se puede ver en la salida obtenida, el número de llamadas es \(log_2(100)=7\), \(log_2(1000)=10\) y \(log_2(10000)=14\), respectivamente, lo que confirma la eficiencia de esta implementación recursiva de potencia.

#include <iostream>

using namespace std;

int NCALLS = 0;

double intpow(double x, int p, bool vrbs=false) {

    NCALLS += 1;
    if (vrbs) {
        cout << "Llamada número " << NCALLS << endl;
    }

    if (p == 1) {
        return x;
    }
    
    if (p == 0) {
        return 1;
    }
    
    if (p % 2 == 0) {
        double z = intpow(x, p / 2, vrbs);
        return z * z;
    } 
    else {
        double z = intpow(x, (p - 1) / 2, vrbs);
        return z * z * x;
    }
}

int main() {
    int base = 1;
    
    NCALLS = 0;
    int exponente = 100;
    cout << "Calculando " << base << "^" << exponente << ":" << endl;
    cout << "Resultado: " << intpow(base, exponente, true) << "\n\n";

    NCALLS = 0;
    exponente = 1000;
    cout << "Calculando " << base << "^" << exponente << ":" << endl;
    cout << "Resultado: " << intpow(base, exponente, true) << "\n\n";

    NCALLS = 0;
    exponente = 10000;
    cout << "Calculando " << base << "^" << exponente << ":" << endl;
    cout << "Resultado: " << intpow(base, exponente, true) << "\n\n";
    
    return 0;
}

Otra forma para demostrar el correcto funcionamiento del código, es utilizar un valor mayor que \(1\), el problema es que al hacer \(x^p\) con \(x>1\) y \(p\) grande, el resultado puede ser tan grande que execeda la capacidad de representación de los números en C++. Para evitar esto, se puede utilizar un valor cercano a \(1\), como por ejemplo \(1,0001\), y sabiendo que \((1+1/p)^p \to e\) cuando \(p \to \infty\). De esta forma, al hacer \(1,0001^{10000}\), se espera obtener un resultado cercano a \(e \approx 2.71828\), lo que permite verificar que la función intpow está calculando correctamente la potencia de un número.

int main() {
    double base = 1.0001;
    
    int exponente = 10000;
    cout << "Calculando " << base << "^" << exponente << ":" << endl;
    cout << "Resultado: " << intpow(base, exponente, true) << "\n\n";
    
    return 0;
}

Ejercicio 2: Similitud

NoteConsigna

Escribir un programa que, dados 4 vectores de enteros \(v1, v2, v3, v4\) busca el primer elemento que está en los cuatro vectores.

Utilizar los vectores almacenados en los archivos f*.txt y escribir una función vector<int> readfile(string filename); que lee los enteros del archivo filename.

Realizar 3 versiones: - GOTO: utilizando goto para salir de los bucles anidados. - BREAK: utilizando break para salir de los bucles anidados. - OPT: versión optimizada del algoritmo, sin utilizar goto ni break.

El primer caso, utilizando GOTO, se implementa mediante 4 bucles anidados, uno por cada vector de enteros. Se posee una etiqueta found a la que se salta mediante goto cuando se encuentra el elemento común en los cuatro vectores (en el bucle más interno).

TipTiempo de ejecución

Para poder comparar el tiempo de ejecución de cada versión, se utiliza la biblioteca <chrono> para medir el tiempo transcurrido desde el inicio hasta el final de la búsqueda del elemento común. Esto permite evaluar la eficiencia de cada implementación y observar las diferencias en el rendimiento entre ellas.

#include <fstream>
#include <string>
#include <vector>
#include <iostream>
#include <chrono>

using namespace std;

vector<int> readfile(string filename) {
    vector<int> values;
    ifstream file(filename);

    int number;
    while (file >> number) {
        values.push_back(number);
    }

    return values;
}


int main() {
    vector<int> values1 = readfile("f1.txt");
    vector<int> values2 = readfile("f2.txt");
    vector<int> values3 = readfile("f3.txt");
    vector<int> values4 = readfile("f4.txt");

    bool FOUND;
    int encontrado = -1;

    auto start = chrono::high_resolution_clock::now();

    for (int v1 : values1) {
        for(int v2 : values2) {
            for(int v3 : values3) {
                for(int v4 : values4) {
                    if (v1==v2 && v2==v3 && v3==v4) {
                        encontrado = v4;
                        goto FOUND;
                    }
                }
            }
        }
    }

    
    FOUND:
    auto end = chrono::high_resolution_clock::now();
    auto duration = chrono::duration_cast<chrono::milliseconds>(end - start);
    cout << "Tiempo de ejecución: " << duration.count() << " ms" << endl;

    if (encontrado != -1) {
        cout << "Valor encontrado: " << encontrado << endl;
    } else {
        cout << "Valor no encontrado" << endl;
    }
    
    return 0;
}

En la segunda versión, utilizando BREAK, se implementa de manera similar a la anterior, pero en lugar de utilizar goto para salir de los bucles anidados, se emplean sentencias break para salir de cada bucle cuando se encuentra el elemento común. Se utiliza una variable booleana FOUND para indicar cuándo se ha encontrado el elemento y así evitar seguir iterando innecesariamente.

Ambos casos son de orden O(n^4) en el peor caso, ya que en el peor escenario se deben comparar todos los elementos de los cuatro vectores entre sí. Difieren principalmente en la legibilidad y las buenas prácticas, siendo generalemente la versión con break más recomendada por evitar el uso de goto, que puede hacer el código más difícil de seguir y mantener.

#include <fstream>
#include <string>
#include <vector>
#include <iostream>
#include <chrono>

using namespace std;

vector<int> readfile(string filename) {
    vector<int> values;
    ifstream file(filename);

    int number;
    while (file >> number) {
        values.push_back(number);
    }

    return values;
}


int main() {
    vector<int> values1 = readfile("f1.txt");
    vector<int> values2 = readfile("f2.txt");
    vector<int> values3 = readfile("f3.txt");
    vector<int> values4 = readfile("f4.txt");

    bool found = false;
    int encontrado = -1;

    auto start = chrono::high_resolution_clock::now();

    for (int v1 : values1) {
        for(int v2 : values2) {
            for(int v3 : values3) {
                for(int v4 : values4) {
                    if (v1==v2 && v2==v3 && v3==v4) {
                        encontrado = v4;
                        found = true;
                        break;
                    }
                }
                if (found) break;
            }
            if (found) break;
        }
        if (found) break;
    }

    auto end = chrono::high_resolution_clock::now();
    auto duration = chrono::duration_cast<chrono::milliseconds>(end - start);
    cout << "Tiempo de ejecución: " << duration.count() << " ms" << endl;

    if (encontrado != -1) {
        cout << "Valor encontrado: " << encontrado << endl;
    } else {
        cout << "Valor no encontrado" << endl;
    }
    
    return 0;
}

Finalmente, la versión optimizada consta de ordenar previamente los cuatro vectores y luego utilizar un enfoque de búsqueda similar al de la intersección de múltiples conjuntos ordenados. Se emplean cuatro índices para recorrer los vectores simultáneamente, avanzando el índice del vector que tenga el valor más pequeño en cada iteración. Este método tiene una complejidad de O(n log n) debido a la necesidad de ordenar los vectores, y luego O(n) para encontrar el elemento común, lo que es significativamente más eficiente que las versiones anteriores.

#include <fstream>
#include <string>
#include <vector>
#include <iostream>
#include <algorithm>
#include <chrono>

using namespace std;

vector<int> readfile(string filename) {
    vector<int> values;
    ifstream file(filename);

    int number;
    while (file >> number) {
        values.push_back(number);
    }

    return values;
}


int main() {
    vector<int> values1 = readfile("f1.txt");
    vector<int> values2 = readfile("f2.txt");
    vector<int> values3 = readfile("f3.txt");
    vector<int> values4 = readfile("f4.txt");

    auto start = chrono::high_resolution_clock::now();

    sort(values1.begin(), values1.end());
    sort(values2.begin(), values2.end());
    sort(values3.begin(), values3.end());
    sort(values4.begin(), values4.end());

    int k1 = 0, k2 = 0, k3 = 0, k4 = 0;
    int encontrado = -1;


    if (values1[0]==values2[0] &&
         values2[0]==values3[0] &&
          values3[0]==values4[0]) {
        encontrado = values1[0];
    } else {
        while (k1<values1.size() && 
                k2<values2.size() &&
                 k3<values3.size() &&
                  k4<values4.size()) {
            if (values1[k1]==values2[k2] &&
                 values2[k2]==values3[k3] &&
                  values3[k3]==values4[k4]) {
                encontrado = values1[k1];
                break;
            }

            int min_value = min({values1[k1], 
                                 values2[k2],
                                 values3[k3],
                                 values4[k4]});

            if (values1[k1] == min_value) k1++;
            if (values2[k2] == min_value) k2++;
            if (values3[k3] == min_value) k3++;
            if (values4[k4] == min_value) k4++;
        }
    }

    auto end = chrono::high_resolution_clock::now();
    auto duration = chrono::duration_cast<chrono::milliseconds>(end - start);
    cout << "Tiempo de ejecución: " << duration.count() << " ms" << endl;

    if (encontrado != -1) {
        cout << "Valor encontrado: " << encontrado << endl;
    } else {
        cout << "Valor no encontrado" << endl;
    }
    
    return 0;
}

Ejercicio 3: Problema de la mochila

NoteConsigna

Implementar una versión análoga al problema de la mochila donde en este caso se poseen \(N\) tareas las cuales requieren una cierta cantidad de tiempo de procesamiento \(t_k\). A su vez, se poseen \(n\) núcleos de computadora donde se pueden distribuir y ejecutar las tareas. El objetivo es encontrar la distribución de tareas que minimice el tiempo total de ejecución.

La implementación propuesta consiste en comenzar ordenando de mayor a menor todos los items iniciales en función de val. De esta forma, no será necesario ir eliminando elementos del vector, sino solo recorrerlo de incio a fin.

Para esto, se implementó una función específica sort_knaps que ordena el vector de items utilizando la función sort de la biblioteca <algorithm>, con un comparador personalizado que ordena los items en función de su valor (val) de mayor a menor.

Luego, por cada elemento de este vector ordenado, se busca el knaps con menor total y se le asigna el item, actualizando su total con el valor del item. De esta forma, se logra una distribución de tareas que minimiza el tiempo total de ejecución, ya que se asignan primero las tareas más costosas a los núcleos menos cargados.

Finalmente, se muestra por pantalla la distribución de tareas en cada knaps junto con el tiempo total de ejecución para cada uno, lo que permite evaluar la eficiencia de la solución propuesta.

#include <fstream>
#include <string>
#include <vector>
#include <iostream>
#include <algorithm>

using namespace std;


struct item_t {
  int id,val;
};

struct knaps_t {
  vector<item_t> items;
  int total;
};

void sort_knaps(vector<item_t>& items) {
  sort(items.begin(), items.end(), [](const item_t& a, const item_t& b) {
    return a.val > b.val;
  });
}

int main() {
  int N=30,nprocs=5,max=10;
  vector<item_t> items;
  printf("items: ");

  for (int k=0; k<N; k++) {
    int val = (rand()%max+1);
    item_t item={k,val};
    items.push_back(item);
    printf("(%d,%d) ",k,val);
  }
  printf("\n");

  vector<knaps_t> knapsv(nprocs);
  for (int k=0; k<nprocs; k++)
    knapsv[k].total = 0;

  //IMPLEMENTACIÓN:
  sort_knaps(items);

  printf("\nitems ordenados: ");
  for (auto item: items)
    printf("(%d,%d) ",item.id,item.val);
  printf("\n");

  for (int i=0; i<N; i++) {
    int min_knaps = 0;
    for (int k=1; k<nprocs; k++) {
      if (knapsv[k].total < knapsv[min_knaps].total)
        min_knaps = k;
    }
    knapsv[min_knaps].items.push_back(items[i]);
    knapsv[min_knaps].total += items[i].val;
  }

  printf("\n");
  for (int k=0; k<nprocs; k++) {
    printf("knaps %d: total=%d items: ",k,knapsv[k].total);
    for (auto item: knapsv[k].items)
      printf("(%d,%d) ",item.id,item.val);
    printf("\n");
  }
}
